# Fruits Classifier CNN

Бинарная классификация изображений «яблоко — апельсин» с использованием свёрточных нейронных сетей (CNN). Проект выполнен на датасете Fruits 360, содержит сравнение двух архитектур и демонстрирует навыки работы с TensorFlow/Keras, предобработкой изображений и оценкой качества моделей.

**Статус:** учебный проект (творческое задание)  
**Университет:** РЭУ им. Г. В. Плеханова, ВШКМиС  
**Дисциплина:** Системы искусственного интеллекта  
**Автор:** Нетиевский Иван Сергеевич  
**Стек:** Python, TensorFlow/Keras, NumPy, Matplotlib, KaggleHub  

## Задача

На основе датасета [Fruits 360](https://www.kaggle.com/datasets/moltean/fruits) построить модель, способную отличать изображения яблок от изображений апельсинов. Решается задача бинарной классификации.

## Датасет

- Используются все изображения классов `Apple*` и `Orange` из тренировочной части Fruits 360.
- Данные разделены на обучающую (80%) и валидационную (20%) выборки.
- Баланс классов: **Apple: 1422 / Orange: 383** в обучающей выборке.

## Архитектуры моделей

### Модель 1 (простая)

Conv2D(32) → MaxPooling2D → Flatten → Dense(128) → Dense(1, sigmoid)


### Модель 2 (углублённая с регуляризацией)
Conv2D(32) → MaxPooling2D →
Conv2D(64) → MaxPooling2D →
Conv2D(128) → MaxPooling2D →
Flatten → Dense(256) → Dropout(0.5) → Dense(1, sigmoid)


## Результаты

| Модель | Val Accuracy | Val Loss |
|--------|--------------|----------|
| Модель 1 | 1.0 | 3.38e-06 |
| Модель 2 | 1.0 | **2.12e-07** |

Обе модели достигли 100% точности на валидационной выборке. Вторая модель показывает значительно меньший уровень ошибки и лучшую уверенность предсказаний.

## Как запустить

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/I1ne/fruits-classifier-cnn.git
   cd fruits-classifier-cnn
   
2. Установите зависимости (рекомендуется Google Colab или локально с GPU):
 ```bash
 pip install tensorflow numpy matplotlib kagglehub
 ```
3. Откройте ноутбук Fruits_360_CNN.ipynb в Jupyter или Google Colab.

4. Запустите ячейки последовательно – датасет загрузится автоматически через kagglehub.

Примечание: для загрузки датасета через kagglehub потребуется файл kaggle.json (API-ключ Kaggle).
